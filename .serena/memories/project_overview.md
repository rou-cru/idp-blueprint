# Project Overview (validated 2025-12-04)

- Purpose: Open-source reference for a compact Internal Developer Platform running on local/lab Kubernetes (k3d by default). Ships GitOps (ArgoCD + ApplicationSets), policy (Kyverno + Policy Reporter), networking (Cilium + Gateway API with wildcard TLS), secrets (Vault + External Secrets Operator), observability (Prometheus Operator CRDs, Grafana, Loki, Fluent Bit, Pyrra), CI/CD (Argo Workflows, SonarQube), and Backstage.
- Deployment: Single command `task deploy` drives the flow defined in `Taskfile.yaml`, reading `config.toml` for versions, NodePorts, passwords, repo/branch overrides, and stack fuses. Stacks behind fuses: policies, observability, cicd, security, backstage; `prod` enables HA for ArgoCD. The events stack is always deployed. Defaults: NodePorts 30080/30443, hostnames `<service>.<lan-ip-dashed>.nip.io`.
- Layout: `IT/` bootstrap layer (k3d cluster config, Cilium, cert-manager, Vault, ESO, ArgoCD, Gateway, namespaces, priority classes); `K8s/` GitOps stacks (ApplicationSets discovery); `Policies/` Kyverno engine + policies; `Taskfile.yaml` + `Task/` orchestration; `Scripts/` helpers (config extraction, vault init/seed, helm-docs, validation); `Docs/` Astro/Starlight site source.
- Tooling/stack: Bash + Taskfile orchestration; Kubernetes manifests/kustomize/helm; Devbox-managed toolchain (kubectl, helm, kustomize, k3d, cilium CLI, argocd CLI, kyverno, helm-docs, yamllint, shellcheck, hadolint, markdownlint, checkov, trufflehog, kubeval, gettext/envsubst, dasel, jq/yq, etc.). Docs build with pnpm + Astro/Starlight (TypeScript). Recommended resources: minimum 4 vCPU / 8 GiB RAM; comfortable 6 vCPU / 12 GiB RAM; ~20 GiB disk free.

---
title: Abbreviations
---

_[IDP]: Internal Developer Platform - A self-service platform for developers to deploy
  and manage applications
_[CNI]: Container Network Interface - Specification and libraries for configuring
  network interfaces in Linux containers
_[GitOps]: Git Operations - Infrastructure and application management using Git as
  single source of truth
_[CI/CD]: Continuous Integration / Continuous Delivery - Automated software delivery
  pipeline
_[RBAC]: Role-Based Access Control - Method of regulating access based on user roles
_[TLS]: Transport Layer Security - Cryptographic protocol for secure communications
_[eBPF]: Extended Berkeley Packet Filter - Linux kernel technology for running
  sandboxed programs
_[YAML]: YAML Ain't Markup Language - Human-readable data serialization language
_[API]: Application Programming Interface - Set of protocols for building software
  applications
_[CRD]: Custom Resource Definition - Extension of Kubernetes API for custom resources
_[SRE]: Site Reliability Engineering - Discipline that incorporates software engineering
  to IT operations
_[SAST]: Static Application Security Testing - Method of debugging by examining source
  code before execution
_[SBOM]: Software Bill of Materials - List of components in a piece of software

_[ArgoCD]: Declarative GitOps continuous delivery tool for Kubernetes
_[Kyverno]: Kubernetes native policy management engine
_[Cilium]: eBPF-based networking, observability, and security solution
_[Vault]: HashiCorp's secrets management and data protection platform
_[Prometheus]: Open-source monitoring and alerting toolkit
_[Grafana]: Multi-platform open-source analytics and visualization platform
_[Loki]: Horizontally scalable log aggregation system
_[Fluent Bit]: Fast and lightweight log processor and forwarder
_[Trivy]: Comprehensive security scanner for vulnerabilities and misconfigurations
_[SonarQube]: Platform for continuous code quality and security inspection
_[K3d]: Lightweight Kubernetes distribution running in Docker
_[K3s]: Lightweight certified Kubernetes distribution
_[Helm]: Package manager for Kubernetes
_[Cert-Manager]: Kubernetes add-on to automate TLS certificate management
_[ESO]: External Secrets Operator - Synchronizes secrets from external APIs to Kubernetes
_[KSM]: Kube State Metrics - Service that listens to Kubernetes API and generates metrics
_[HPA]: Horizontal Pod Autoscaler - Automatically scales number of pods
_[VPA]: Vertical Pod Autoscaler - Automatically adjusts CPU and memory requests
_[OPA]: Open Policy Agent - Policy-based control for cloud native environments
_[mTLS]: Mutual TLS - Two-way authentication using TLS certificates
_[SPIFFE]: Secure Production Identity Framework For Everyone
_[SPIRE]: SPIFFE Runtime Environment - Production-ready implementation of SPIFFE
_[CRI]: Container Runtime Interface - Plugin interface for container runtimes
_[CSI]: Container Storage Interface - Standard for exposing storage systems to containers
_[Ingress]: Kubernetes resource that manages external access to services
_[NetworkPolicy]: Kubernetes specification for controlling network traffic between pods
_[Service Mesh]: Infrastructure layer for handling service-to-service communication
_[Observability]: Ability to measure internal states of a system by examining outputs
_[Telemetry]: Automated collection and transmission of data from remote sources
_[IaC]: Infrastructure as Code - Managing infrastructure through machine-readable files
_[Policy as Code]: Managing security and compliance policies through version-controlled
  code

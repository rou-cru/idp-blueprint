House style for authoring documentation.

## Principles

- Short sections, task-oriented titles
- Use admonitions (`note`, `tip`, `warning`) sparingly but effectively
- Prefer diagrams for flows; one objective per diagram

## Formatting

- Headings: sentence case; concise
- Code blocks with language hints; enable copy buttons
- Use tabs/cards for alternative paths or personas

# Abbreviations

*[IDP]: Internal Developer Platform - A self-service platform for developers to deploy and
  manage applications
*[CNI]: Container Network Interface - Specification and libraries for configuring
  network interfaces in Linux containers
*[GitOps]: Git Operations - Infrastructure and application management using Git as
  single source of truth
*[CI/CD]: Continuous Integration / Continuous Delivery - Automated software delivery
  pipeline
*[RBAC]: Role-Based Access Control - Method of regulating access based on user roles
*[TLS]: Transport Layer Security - Cryptographic protocol for secure communications
*[eBPF]: Extended Berkeley Packet Filter - Linux kernel technology for running sandboxed
  programs
*[YAML]: YAML Ain't Markup Language - Human-readable data serialization language
*[API]: Application Programming Interface - Set of protocols for building software
  applications
*[CRD]: Custom Resource Definition - Extension of Kubernetes API for custom resources
*[SRE]: Site Reliability Engineering - Discipline that incorporates software engineering to
  IT operations
*[SAST]: Static Application Security Testing - Method of debugging by examining source
  code before execution
*[SBOM]: Software Bill of Materials - List of components in a piece of software

*[ArgoCD]: Declarative GitOps continuous delivery tool for Kubernetes
*[Kyverno]: Kubernetes native policy management engine
*[Cilium]: eBPF-based networking, observability, and security solution
*[Vault]: HashiCorp's secrets management and data protection platform
*[Prometheus]: Open-source monitoring and alerting toolkit
*[Grafana]: Multi-platform open-source analytics and visualization platform
*[Loki]: Horizontally scalable log aggregation system
*[Fluent Bit]: Fast and lightweight log processor and forwarder
*[Trivy]: Comprehensive security scanner for vulnerabilities and misconfigurations
*[SonarQube]: Platform for continuous code quality and security inspection
*[K3d]: Lightweight Kubernetes distribution running in Docker
*[K3s]: Lightweight certified Kubernetes distribution
*[Helm]: Package manager for Kubernetes
*[Cert-Manager]: Kubernetes add-on to automate TLS certificate management
*[ESO]: External Secrets Operator - Synchronizes secrets from external APIs to Kubernetes
*[KSM]: Kube State Metrics - Service that listens to Kubernetes API and generates metrics
*[HPA]: Horizontal Pod Autoscaler - Automatically scales number of pods
*[VPA]: Vertical Pod Autoscaler - Automatically adjusts CPU and memory requests
*[OPA]: Open Policy Agent - Policy-based control for cloud native environments
*[mTLS]: Mutual TLS - Two-way authentication using TLS certificates
*[SPIFFE]: Secure Production Identity Framework For Everyone
*[SPIRE]: SPIFFE Runtime Environment - Production-ready implementation of SPIFFE
*[CRI]: Container Runtime Interface - Plugin interface for container runtimes
*[CSI]: Container Storage Interface - Standard for exposing storage systems to containers
*[Ingress]: Kubernetes resource that manages external access to services
*[NetworkPolicy]: Kubernetes specification for controlling network traffic between pods
*[Service Mesh]: Infrastructure layer for handling service-to-service communication
*[Observability]: Ability to measure internal states of a system by examining outputs
*[Telemetry]: Automated collection and transmission of data from remote sources
*[IaC]: Infrastructure as Code - Managing infrastructure through machine-readable files
*[Policy as Code]: Managing security and compliance policies through version-controlled code

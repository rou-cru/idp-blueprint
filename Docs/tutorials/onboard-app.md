# Tutorial: Onboard an Application

End-to-end walkthrough to onboard a sample application using the GitOps model.

## Goals

- Create a new Kustomize overlay and Helm values
- Register via ApplicationSet
- Validate deployment and observability

## Prerequisites

- Completed [Quickstart](../getting-started/quickstart.md)
- Access to the repository fork/branch

## Steps

1) Scaffold directories and values
2) Add labels and canonical metadata
3) Commit and push; ArgoCD reconciles
4) Verify logs/metrics and routes
